# Initialize Project Primer

Analyze the current project and generate a comprehensive primer for Claude Code sessions.

## Instructions

### Step 1: Check for Existing Primer

Check if `.claude/PRIMER.md` exists:
- If it exists and `$ARGUMENTS` does NOT contain `--force`, ask: "Primer already exists. Run `/session-manager:init --force` to regenerate."
- If `--force` is passed or no primer exists, continue with generation.

### Step 2: Gather Project Information

Run these commands in parallel to understand the project:

```bash
# Package manager and dependencies
ls package.json pnpm-lock.yaml yarn.lock package-lock.json bun.lockb Cargo.toml go.mod requirements.txt Gemfile pyproject.toml 2>/dev/null

# Git info
git remote -v 2>/dev/null
git branch --show-current 2>/dev/null

# Directory structure (top-level)
ls -la

# Look for existing docs
ls README.md CONTRIBUTING.md docs/ .claude/ 2>/dev/null
```

### Step 3: Detect Tech Stack

Based on files found, identify:

| File | Indicates |
|------|-----------|
| `package.json` | Node.js project |
| `pnpm-lock.yaml` | Uses pnpm |
| `yarn.lock` | Uses yarn |
| `tsconfig.json` | TypeScript |
| `vite.config.*` | Vite bundler |
| `next.config.*` | Next.js |
| `wxt.config.*` | WXT browser extension |
| `tailwind.config.*` | Tailwind CSS |
| `Cargo.toml` | Rust |
| `go.mod` | Go |
| `pyproject.toml` / `requirements.txt` | Python |
| `.env.example` | Environment variables needed |

Read the main config file (package.json, Cargo.toml, etc.) to extract:
- Project name
- Scripts/commands available
- Key dependencies

### Step 4: Analyze Project Structure

Identify the architecture pattern:
- **Monorepo**: Multiple `package.json` files, workspaces config
- **Frontend app**: `src/components/`, `src/pages/`, `src/app/`
- **Backend API**: `src/routes/`, `src/controllers/`, `src/handlers/`
- **Library**: `src/lib/`, `src/index.ts` as entry
- **Full-stack**: Both frontend and backend directories
- **Browser extension**: `manifest.json`, `background/`, `content/`, `popup/`

### Step 5: Find Existing Conventions

Look for patterns in the codebase:
- Naming conventions (camelCase, snake_case, kebab-case)
- File organization patterns
- Common imports/utilities
- Testing patterns (`*.test.ts`, `*.spec.ts`, `__tests__/`)

Read a few representative files to understand code style.

### Step 6: Generate the Primer

Create `.claude/PRIMER.md` with this structure:

```markdown
# Project Primer: [Project Name]

> Auto-generated by `/session-manager:init` on [DATE]
> Regenerate with `/session-manager:init --force`

## Quick Start

```bash
[package manager] install    # Install dependencies
[package manager] dev        # Start development
[package manager] build      # Build for production
[package manager] test       # Run tests
```

## Tech Stack

| Category | Technology |
|----------|------------|
| Language | [TypeScript/JavaScript/Rust/Go/Python] |
| Framework | [React/Vue/Next.js/Express/etc.] |
| Styling | [Tailwind/CSS Modules/styled-components] |
| Package Manager | [pnpm/npm/yarn/cargo/go] |
| Build Tool | [Vite/Webpack/esbuild/etc.] |
| Testing | [Jest/Vitest/pytest/etc.] |

## Project Structure

```
[project-name]/
├── src/           # [description]
├── tests/         # [description]
├── ...
```

## Key Directories

- **[dir]**: [what it contains and when to look here]

## Architecture Notes

[Brief description of the architecture pattern]

## Common Tasks

### Adding a new [feature/component/route]
[Steps based on existing patterns]

### Running specific tests
```bash
[test command]
```

### Building for production
```bash
[build command]
```

## Conventions

- **File naming**: [pattern observed]
- **Component structure**: [pattern observed]
- **Import ordering**: [pattern observed]

## Environment Variables

[List from .env.example if exists]

## Important Files

- `[file]` - [why it's important]

## External Resources

- [Link to docs if detected]
```

### Step 7: Create Supporting Files (if needed)

If `.claude/` directory doesn't exist, create it.

If useful patterns are detected, optionally create:
- `.claude/ARCHITECTURE.md` - Deeper architecture notes
- `.claude/COMMON_TASKS.md` - Step-by-step guides for common operations

### Step 8: Output Summary

```
Primer generated: .claude/PRIMER.md

Detected:
- Tech stack: [summary]
- Architecture: [type]
- Package manager: [name]

Key commands:
- [dev command]
- [build command]
- [test command]

To view the full primer:
  Read .claude/PRIMER.md

To regenerate after major changes:
  /session-manager:init --force
```

## Important Notes

- Be thorough but concise - this will be read at the start of sessions
- Focus on actionable information, not obvious things
- Include gotchas or non-obvious patterns you notice
- If monorepo, document each workspace briefly
- Don't include sensitive information (API keys, secrets)
